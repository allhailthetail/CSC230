** The Call Stack
- *JavaScript is at its core a synchronous (single threaded) language. However, JavaScript can mimic asynchronous execution in certain situations using the event loop and task queue.*
  When you call a public API using JavaScript, it can take anywhere from a few milliseconds to several seconds to receive the data. If JavaScript were purely synchronous, your entire
  site would freeze until the data was returned. That‚Äôs unacceptable for real-world applications. Using asynchronous patterns like async/await or .then(), you can allow your code to
  continue running while waiting for the data and handle it as soon as it‚Äôs available.

- The call stack is a LIFO queue and runs the *synchronous* code in LIFO order. Any Asynchronous code is handed off to the browser, which can deal with these pieces in the background...
  Once the asynch pieces are done, they're passed back to the call stack?

- Browser APIs:
  In this context, API is actually functionality that your own browser (its under-the-hood C++ code, etc) is exposing to
  the very simple JavaScript engine...

  #+begin_example js
  console.log("Start");  // Executed synchronously

  setTimeout(() => {    // Executed asynchronously so that other things can proceed...
      console.log("Timeout done");
  }, 1000);

  console.log("End");   // Executed synchronously while the middle line is in the background...

  // Notice how confusing this'd be if you didn't know what was actually happening:
  // Start
  // End
  // Timeout done
  #+end_example

** Sending API Requests:
- Option 1: async/await
  - Relies on a system of *promises:*
    * Pending: Initial state awaiting a resolution of an asynchronous operation
    * Fulfilled: Result of a successful asynchronous operation.
    * Rejected: Result of an unsuccessful asynchronous operation.

  - [[https://www.w3schools.com/jsref/api_fetch.asp][W3S Example]]
    #+begin_example js
    // getDogFact() code not shown
    const button = document.getElementById("fetch-dog-btn");

    button.addEventListener("click", function () {
        getDogFact();
    });
    #+end_example

  - Uses fetch() to request data from the Dog API and then awaits the response.
  - Converts the response to readable JSON using .json() and we also await here
    since it requires waiting for the Promise to be resolved.
  - Gets the dog fact and prints it to the console.
  - Updates the DOM with the returned dog fact

- Option 2: .then()
  - A simpler option...
    * A success callback
    * An error callback.
  - Example:
    #+begin_example js
    function getDogFact() {
      fetch("https://dogapi.dog/api/facts")
      .then(response => response.json())  // Covert the response
      .then(data => {
        // Update the DOM with the dog fact
        const fact = data.facts[0];
        const output = document.getElementById("fact-output");
        output.textContent = fact;
      })
      .catch(error => {
        // Message to print if there was an error
        console.error("Error fetching dog fact:", error);
      });
    }

    // Same button code as async/await
    const button = document.getElementById("fetch-dog-btn");

    button.addEventListener("click", function () {
      getDogFact();
    });
    #+end_example
